# Universe of Finance — Transaction Taxonomy

> A living catalogue of every financial transaction type in the global economy,
> organised into sectors, categories, and subcategories.

This taxonomy is the backbone of the research. Every category gets its own
research track with independent data collection, analysis, and projections.

## Double-Counting Rules

To avoid inflating the total:

1. **Settlement vs. trading**: A stock trade is ONE transaction, even if it
   triggers clearing, settlement, and custody movements
2. **Intermediary hops**: A remittance is ONE transaction, even if it passes
   through correspondent banks
3. **Netting**: Where systems net transactions, count the gross number of
   original transactions, not the netted settlement count
4. **Conversions**: A payment that involves FX conversion counts as ONE payment
   transaction plus ONE FX transaction (different categories)
5. **DeFi composability**: A DeFi swap through an aggregator that routes through
   3 pools counts as the number of on-chain transactions executed

## Adding New Categories

When the Scout or Architect identifies a new transaction type:

1. Find the appropriate sector (or propose a new one)
2. Add the category with description and key sources
3. Update this file via PR
4. The Architect will design a `METHODOLOGY.md` for the new category
